Web Page Layout: A Brief History

Back in the early days of the World Wide Web, brave and tech-savvy graphic designers embarked on a journey into uncharted territory: applying design theory to a new digital medium. These were the days before Cascading Style Sheets, when HTML alone was used to both markup content and stylistically embellish a page's visual presentation.

This was also a time before HTML had structural elements such as header, article, section, aside, and footer. In fact, even the generic div element had yet to be concieved. Web designers needed something to contain areas of a page layout, so they took to using HTML tables as layout containers — and blatantly defied the table element's intended use to serve their own aesthetic purposes.

Evolution came and brought us the div and span elements, designed to contain areas of content and be styled with CSS. This simplified the way that design could be applied to web pages while also allowing for a more versatile separation of structure and presentation. CSS properties like float and position were utilized to shape a page's elements into various layout configurations. Still, there was room for improvement as patterns emerged and the need for specific types of containers became apparent. CSS layout methods weren't super smooth either and had some common issues that became apparent as designers sought to reproduce common layouts and innovate creative new solutions to emerging problems brought by changing technology.

We now have HTML elements that are specifically intended to contain portions of a page's content and to serve as foundational structure for page layout. We also have CSS capabilities that allow us more control than ever while also smoothing out some long-standing issues and, finally, giving us a full-fledged system for establishing a grid foundation for page layout.

Of course, things will continue to evolve for HTML and CSS. Whatever the future will hold, it's bound to be interesting!

Layout Defaults: Naturally Flowing Blocks

Flowing Blocks? Whaaaa? While you may not think of "blocks" as something "natural" or "flowing", in a modern, unstyled HTML page that's exactly what we have as a default. Whether it's simply headings and paragraphs or if the page's content is further defined and organized with header, nav, section, aside, footer and other structural elements, we have by default a set of stacked block elements.

These block elements are what we manipulate with CSS in order to give a page its visual appearance and form. By default, the "natural flow" of a page's block elements is a vertical stack of content. Each block has a height that's defined by the content it contains, and a width that expands to the boundaries of the page body.

Moving Things Around With Position

Sometimes you just gotta move an element into a different spot or change its position in relation to other elements. That's where the position property comes in. There are five possible position values currently available in CSS:

  • static - the default position. Static positioned elements aren't affected by the top, bottom, left, and right properties. Once an element's position is defined through CSS as something other than static, the element is considered to be "positioned".
  • relative - allows positioning relative to the element's normal position in the document flow. Elements with relative position can be moved using the top, bottom, left, and right properties.
  • absolute - allows positioning using specific coordinates for the top, bottom, left, and right properties that change the position of those respective sides of the element. Coordinates are relative to the element's nearest ancestor that has a position other than static. If no positioned ancestor exists, absolute coordinates are relative to the document body. Absolute-positioned elements are still connected to the document's flow and will scroll with the rest of the page.
  • fixed - allows positioning of an element's top, bottom, left, and right sides relative to the browser viewport. Fixed-position elements are completely removed from the document's flow and do not scroll along with the content of the page.
  • sticky - this type of positioning is like a hybrid of relative and fixed positioning. An element with sticky positioning acts as a relative-positioned element until a particular offset is reached when scrolling the page's content. Once that offset is reached, the element becomes fixed and "sticks" in place.
